Read the backend development guidelines before starting your development task.

Execute these steps:
1. Read `.trellis/spec/backend/index.md` to understand available guidelines
2. Based on your task, read the relevant guideline files:
   - Protocol and phase order → `.trellis/spec/backend/00-protocol.md`
   - Workflow map → `.trellis/spec/backend/10-workflow/index.md`
   - Model work → `.trellis/spec/backend/20-checklists/model-checklist.md`, `.trellis/spec/backend/database.md`
   - Service work → `.trellis/spec/backend/20-checklists/service-checklist.md`, `.trellis/spec/backend/security.md`
   - API work → `.trellis/spec/backend/20-checklists/api-checklist.md`, `.trellis/spec/backend/api-core.md`, `.trellis/spec/backend/routing.md`
3. Understand the coding standards and patterns you need to follow
4. Before concluding, run `pnpm backend:workflow`
5. Then proceed with your development plan

This step is **mandatory** before writing any backend code.
