# Kooboo 通用交付检查清单

在交付 Kooboo 相关代码前，请先对照本清单（通用部分），再对照当前模式的清单（SSR / Script Vue / Local Vue）。

---

## 1. 页面和路由


| 检查项     | 说明                                           | 状态  |
| ------- | -------------------------------------------- | --- |
| 页面有路由定义 | 每个 `.html` 页面文件顶部必须有 `<!-- @k-url /path -->` | [ ] |
| 路由格式正确  | 动态参数使用 `{param}` 格式，如 `/product/{id}`        | [ ] |
| 路由不重复   | 检查是否有多个页面定义相同路由                              | [ ] |
| 静态路由在前  | 静态路由应放在动态路由前面                                | [ ] |


---

## 2. API 定义


| 检查项             | 说明                                        | 状态  |
| --------------- | ----------------------------------------- | --- |
| 正确的请求类型         | GET 用 `k.api.get()`，POST 用 `k.api.post()` | [ ] |
| 错误处理            | 返回适当的 HTTP 状态码 (400, 401, 404, 500)       | [ ] |
| 参数验证            | 验证必要参数是否存在                                | [ ] |
| 输入 sanitization | 对用户输入进行过滤和转义                              | [ ] |
| **@k-url 声明**   | API 文件顶部必须有 `// @k-url /api/xxx/{action}` | [ ] |
| **Model 引用路径**  | 有聚合用聚合路径（如 `code/Models/index`），无聚合用 `code/Models/模型名`；禁止 `code/Models` | [ ] |
| **Services 位置**  | 业务逻辑在 `code/Services/xxx.ts`，不在 `code/` 根目录 | [ ] |
| **API 按模块合并**  | 同一模块的多个接口放在一个 api 文件中 | [ ] |
| **单资源 id**   | 类似 id 的参数使用 query（如 `?id=`），避免路径 `/{id}` | [ ] |
| **Token 编解码**   | 使用 `k.security.jwt.encode/k.security.jwt.decode` 时，decode 返回格式为 `{code, value}` | [ ] |
| **同步操作**        | k_sqlite 是同步的，不需要 `await`                 | [ ] |


---

## 3. 数据库操作

### k.DB.sqlite


| 检查项      | 说明                | 状态  |
| -------- | ----------------- | --- |
| SQL 注入防护 | 使用参数化查询，不要拼接字符串   | [ ] |
| 错误处理     | 数据库操作必须 try-catch | [ ] |


### k_sqlite ORM


| 检查项           | 说明                            | 状态  |
| ------------- | ----------------------------- | --- |
| 模型时间戳         | 使用 `timestamps: true` 启用自动时间戳 | [ ] |
| 索引            | 常用查询字段添加 `index: true`        | [ ] |
| 分页            | 大数据量查询使用 `findPaginated`  | [ ] |
| 复杂查询            | 复杂查询使用 `k.DB.sqlite.query`, 用原生 sql 查询  | [ ] |
| **不使用 await** | k_sqlite 是同步操作，不需要 `await`    | [ ] |
| **使用 _id**    | 主键字段是 `_id`，不是 `id`           | [ ] |

## 4. 安全性


| 检查项    | 说明                                 | 状态  |
| ------ | ---------------------------------- | --- |
| 密码加密   | 存储密码使用 `k.security.md5` 或更强算法 | [ ] |
| 敏感数据保护 | 不要在客户端暴露敏感信息                       | [ ] |
| 输入验证   | 所有用户输入必须验证                         | [ ] |
| SQL 注入 | 使用参数化查询                            | [ ] |
| XSS 防护 | 输出时进行 HTML 转义                      | [ ] |
